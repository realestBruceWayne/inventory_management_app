# Deployment Guide

This guide provides step-by-step instructions for deploying the Inventory & Order Management System to free hosting platforms.

## Prerequisites

- GitHub account with the repository pushed
- Render account (for backend)
- Vercel account (for frontend)

## Backend Deployment (Render)

### Option 1: Using Render Dashboard (Recommended)

1. **Create Render Account**
   - Go to [render.com](https://render.com) and sign up/login
   - Connect your GitHub account

2. **Deploy the Backend**
   - Click "New +" → "Web Service"
   - Connect your GitHub repository
   - Select the repository branch (master)
   - Render will automatically detect the `render.yaml` configuration
   - Review the configuration:
     - Name: inventory-management-api
     - Environment: Python
     - Plan: Free
     - Build Command: `pip install -r requirements.txt`
     - Start Command: `uvicorn backend.main:app --host 0.0.0.0 --port $PORT`
   - Render will automatically create the PostgreSQL database
   - Click "Deploy Web Service"

3. **Get Backend URL**
   - After deployment, Render will provide a URL like: `https://inventory-management-api.onrender.com`
   - Save this URL for frontend configuration

### Option 2: Using Render CLI

```bash
# Install Render CLI
npm install -g render-cli

# Login
render login

# Deploy
render deploy
```

## Frontend Deployment (Vercel)

### Option 1: Using Vercel Dashboard (Recommended)

1. **Create Vercel Account**
   - Go to [vercel.com](https://vercel.com) and sign up/login
   - Connect your GitHub account

2. **Deploy the Frontend**
   - Click "Add New..." → "Project"
   - Import your GitHub repository
   - Configure the project:
     - Framework Preset: Vite
     - Root Directory: `frontend/frontend`
     - Build Command: `npm run build`
     - Output Directory: `dist`
   - Add Environment Variables:
     - `VITE_API_URL`: Your backend URL from Render (e.g., `https://inventory-management-api.onrender.com`)
   - Click "Deploy"

3. **Get Frontend URL**
   - After deployment, Vercel will provide a URL like: `https://your-project.vercel.app`

### Option 2: Using Vercel CLI

```bash
# Install Vercel CLI
npm install -g vercel

# Login
vercel login

# Deploy from frontend directory
cd frontend/frontend
vercel --prod

# Set environment variable
vercel env add VITE_API_URL production
```

## Environment Variables

### Backend (Render)
These are automatically configured by render.yaml:
- `DATABASE_URL`: Auto-generated from Render PostgreSQL
- `SECRET_KEY`: Auto-generated by Render
- `API_KEY`: Auto-generated by Render

### Frontend (Vercel)
Must be manually configured:
- `VITE_API_URL`: Your deployed backend URL (e.g., `https://inventory-management-api.onrender.com`)

## Verification Steps

1. **Backend Health Check**
   ```bash
   curl https://your-backend-url.onrender.com/health
   ```
   Should return: `{"status":"healthy","database":"connected"}`

2. **API Documentation**
   - Visit: `https://your-backend-url.onrender.com/docs`
   - Should show FastAPI documentation

3. **Frontend Access**
   - Visit your Vercel URL
   - Should show the login page
   - Test with credentials: `admin` / `admin123`

## Troubleshooting

### Backend Issues
- Check Render logs for build errors
- Ensure `requirements.txt` is in the root directory
- Verify the start command points to the correct module

### Frontend Issues
- Ensure `VITE_API_URL` is set correctly in Vercel
- Check that the backend URL is accessible
- Verify CORS settings allow your frontend domain

### Database Issues
- Render automatically creates and connects the database
- Check database is not sleeping (free tier spins down after inactivity)
- First request may take longer due to cold start
